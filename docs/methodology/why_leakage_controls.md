# Why Leakage Controls

Target leakage occurs when predictors contain the answer the model is supposed to learn. In this project, `risk_label` is derived from procedure risk, bid count, buyer dependency, and amendments. Feeding those fields directly into the classifier would inflate metrics and weaken dissertation validity.

The active predictor set excludes label-defining fields and tests enforce that policy. This makes performance harder to achieve but more academically defensible.

Leakage controls are also applied conceptually in supplier behavior features: historical supplier attributes should not use information dated after the focal contract.
